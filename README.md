# SOC 合规自动化软件重写后端

## 当前目标

这是新的正式后端工程，目标是替代旧版 `backend`，统一以下基线：

- 编码统一使用 `UTF-8`
- 文档说明以中文为主
- 领域模型按 `项目 -> 请求 -> RCM` 主链路重建
- 旧版代码只作为参考，不再继续叠加历史结构

## 当前已搭建模块

- 认证与 JWT 基础设施
- 项目管理
- 请求管理
- RCM 管理
- 核心建表 SQL

## 启动方式

1. 初始化数据库，执行 `src/main/resources/sql/001_init_core_schema.sql`
2. 按需修改 `src/main/resources/application.yml`
3. 启动命令：

```bash
mvn spring-boot:run
```

## 说明

- 第一版优先保证工程结构清晰、可编译、字段统一
- 文件上传先走本地存储，后续可切换到对象存储
- 控制测试、差距分析、评分、报表模块会在下一阶段继续补齐
